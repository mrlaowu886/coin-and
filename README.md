一、执行摘要
Coinand 是基于 BSC（币安智能链）区块链构建的去中心化裂变增长协议，旨在通过 “用户价值” 的创新模式，将用户的社交网络转化为可量化的增长动力。项目核心在于运用 BSC 的卓越性能（包括低廉的 Gas 费用、高吞吐量 TPS）以及智能合约地址映射技术，打造一套透明度高、效率出众、可验证的多种用户关系奖励机制，从而应对 Web3 项目面临的用户增长瓶颈、激励机制模糊以及网络效应不足等挑战。
通过将用户关系链上化、奖励规则算法化、数据看板可视化，Coinand 重新定义了 Web3 时代的用户增长逻辑，让每个用户都能通过社交网络的扩张获得实实在在的价值回报，同时为项目方提供低成本、高转化的增长解决方案。
二、项目背景与痛点
2.1 Web3 增长困境
当前 Web3 项目普遍面临用户获取成本高、留存率低、网络效应难以形成等问题：
传统邀请机制低效：中心化平台的用户行为激励规则不透明，存在篡改数据、克扣奖励等风险；
激励断层问题显著：用户邀请行为未能获得相应的价值回报，导致持续参与动力不足；
防作弊难：女巫攻击（Sybil Attack）普遍，虚假行为消耗项目资源；
跨平台割裂：不同项目的用户体系独立，用户社交价值无法跨生态复用。
2.2 BSC 生态的机遇
BSC 作为高性能公链，兼容以太坊虚拟机（EVM），拥有每秒数千笔交易（TPS）、秒级确认时间和极低的 Gas 费（约 0.000005 BNB / 笔），为构建实时、低成本的用户行为贡献奖励系统提供了技术基础。此外，BSC 成熟的智能合约生态和地址映射技术，可实现链上确定性关系存储，为邀请网络的可信构建提供了原生支持，同时其庞大的用户基数和丰富的生态应用，也为项目落地提供了海量流量基础。
三、核心价值主张
1.链上信任机制：基于 BSC 智能合约构建不可篡改的用户关系链，所有用户行为和奖励记录均可通过链上数据验证；
2.病毒式激励机制：我们设计了多维度奖励体系，确保每位参与者都能根据其贡献获得相应奖励，从而最大化社交网络的整体价值；
3.实时透明：积分计算和奖励发放实时上链，用户可通过数据看板实时追踪收益，杜绝暗箱操作；
4.动态身份凭证：注册成功将可以实现项目共建，实时展示积分、排名等数据，兼具社交传播性和收藏价值。
四、技术架构
4.1 底层技术栈
区块链：BSC 主网（兼容测试网 BSC Testnet）
开发框架：Hardhat/Remix（BSC 智能合约主流开发框架，EVM 兼容）
存储方案：采用 BSC 智能合约地址映射+链上存储，实现用户关系及积分数据的安全存证
钱包支持：支持MetaMask、Trust Wallet、TokenPocket、SafePal等主流BSC钱包
4.2 核心模块设计
（1）用户账户系统
Solidity
Solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;
contract UserProfile {
    address public authority;       // 用户钱包公钥
    string public invite_code;     // 专属识别码（基于钱包地址哈希生成）
    uint64 public points;             // 累计积分
    address public referrer; // 用户推荐人地址（无则为address(0)）
    uint64 public created_at;         // 注册时间戳
    uint64 public last_active;        // 最后活跃时间戳
}
每个用户对应一个链上账户，通过钱包地址唯一标识；
内置防女巫攻击机制：通过结合 Gas 费验证来确保账户的真实付费行为，并利用设备指纹技术限制单设备创建多个账户。
（2）用户关系链
基于 BSC 智能合约地址映射和 Merkle Tree 构建多维度用户关系：
其邀请关系通过referrer字段直接存储在链上；
通过 Merkle Proof 压缩存储，降低链上存储成本；
关系不可篡改：一旦用户通过确认关系注册，用户关系永久写入链上，支持链上验证。
（3）积分结算引擎
实时积分计算：当用户完成注册、宣发攻坚贡献等行为时，智能合约将自动触发积分计算流程，确保在1秒内完成链上的积分确认。
奖励公式：
L1 奖励 = 基础积分（10）× 被邀请人活跃度系数
L2 奖励 = 基础积分（5）× 二级用户邀请效率系数
L3 奖励 = 基础积分（2）× 三级用户网络贡献系数
防刷机制：设置 “有效用户关系” 门槛（如用户行为人完成至少 1 次链上交互），避免女巫获利。
（4）动态 NFT 生成器
用户的元数据（积分、排名、邀请数）实时关联链上数据，通过预言机同步更新；
视觉权重设计：按照用户关注度优先级分配展示区域（参考饼图数据）：
实时积分数字占比30%，用户专属二维码占比25%，动态进度条占比20%，项目LOGO占比15%，BSC链标识占比10%。
五、产品功能
5.1 极简注册流程
支持用户通过钱包实现一键快速登录，省去了繁琐的表单填写步骤；
自动生成唯一标识，绑定用户钱包地址；
BSC的注册成本极低，大约只需支付0.0005 BNB，且交易确认速度极快，通常在1秒内即可完成链上确认。
5.2 多纬度奖励体系
层级	奖励系数	触发条件	价值捕获
L1 直接邀请	10 积分	被邀请人完成注册 + 首次交互	基础奖励 + 被邀请人后续贡献的 10%
L2 间接邀请	5 积分	二级用户产生有效邀请	二级用户奖励的 50%
L3 网络效应	2 积分	三级用户活跃度达标（≥3 次交互）	三级用户奖励的 20%
5.3 可视化数据看板
个人中心将展示用户的累计积分、以3D力导向图呈现的邀请图谱，以及实时排名信息（如‘TOP 10’）。
趋势图表：积分增长曲线、邀请转化漏斗、网络扩展速度。
5.4 动态积分身份
生成：用户注册后自动 mint 10 积分，作为用户身份凭证；
特性：实时显示积分变化、排名升降、邀请进度，支持社交平台直接分享；
权益：持有积分越多级别越高，奖励比例也越高（如token 分成比例提升）。
六、积分模型
6.1 积分（Coinand Points, CP）设计
总量：初始发行的积分，不可流通，仅作为用户自身在Coinand 贡献值的凭证，非可交易的代币
使用场景：
可用于兑换BSC生态项目白名单，包括新币发行额度及NFT铸造资格等。
可获得治理权，参与协议参数的调整投票。
未来可通过 DAO 投票决定是否映射为流通型代币。
6.2 价值支撑
项目方需支付推广预算给Coinand，该预算用于补贴用户奖励（以BNB/BSC生态主流代币结算）。
生态分成：用户通过带来的项目交易手续费，按比例返还积分奖励；
未来可能会发行 NFT ，NFT 增值：动态 NFT 的稀缺性（基于用户网络规模）可带来收藏价值。
七、生态合作
Coinand 采用开放生态策略，欢迎三类伙伴合作：
1.BSC DApp：接入 Coinand 的用户系统，低成本获取真实用户；
2.钱包 / 工具：集成 Coinand SDK，为用户提供一站式用户体验；
3.NFT 项目 / 创作者：联合发行限定版 NFT，扩大传播范围。
八、结语
Coinand 的使命是让每个用户的社交网络都能产生可量化的价值，致力于让Web3项目的成长回归至‘用户为核心’的本质。通过 BSC 的技术赋能和透明的激励机制，我们相信 Coinand 将成为连接用户与项目的核心枢纽，推动 Web3 生态从 “流量争夺” 走向 “网络共建”。
